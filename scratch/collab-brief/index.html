Design handoff · Collab Platform

A visual identity for the workspace, not another Discord clone.

The product is fully built and working — chat, tasks, notes, calls, the lot. What it's missing is a look. This brief lays out every screen and state that needs designing, and the system to hand back so it drops straight into code.

You design · no code needed Audience: small teams, 3–10 The task board is the star Currently dark-only

Direction No dark indigo.

The current build leans on #4f46e5 indigo everywhere — the Discord-ish blurple. That's exactly what we're moving away from. Pick a fresh accent that isn't indigo, purple, or Discord-blue.

Everything else about the palette is wide open. The accent, the surfaces, the whole thing — your call.

01

What Collab is

One workspace for a small team to chat, track tasks, keep notes, and hop on a call — instead of juggling Discord + Trello + Google Docs + Zoom.

  • Audience: small teams of ~3–10 (student teams, tiny startups) who want one tool, not five open tabs.
  • Feel we're after: focused, calm, a bit sharp. A tool you sit inside all day.
  • The shape: Discord's layout, but the task board is the reason to be here. Give it the most polish.

03

Screens & surfaces to design

Twenty-five surfaces, grouped. Each already exists in the product — you're skinning it, not inventing it. Tags mark the heavy hitters and the real gaps.

A Getting in
Landing page

Public, logged-out marketing page: hero, a "start at your desk" callout, a grid of ~9 feature cards, closing CTA, footer.

Needs: hero styling, feature-card style, CTA buttons.

Login

Email sign-in (magic link / OAuth). Small, centered.

Needs: the auth card.

Onboarding

New user picks a display name and claims a unique @username.

Needs: the form + "username taken / available" feedback states.

Join by invite

Someone opened an invite link — a confirm-and-join screen.

States: valid invite · expired/invalid · already a member.

B The Desk — home dashboard
The Deskfirst thing you see

Three stacked sections: Waiting on you (tasks assigned to you across all teams, with due labels like due in 3h / 2d overdue) · Your teams (cards with avatar, member count, open-task count, unread badge) · Direct messages (with an all / unread filter).

Needs: section-header style, the waiting-task row, the team card, the DM row, empty states, unread badge, and the overdue / urgent treatment.

C App shell
Rail

Round 44px icon buttons: active / idle / hover, unread dot on a team, red count badge on DMs, the "+" new-team button, avatar + "Log out" at the foot.

Needs: the icon-button system + active / unread indicators.

Channel column

Team header, invite button + invite-link popover (with a "copied!" state), channel rows (icon + name, active/idle/hover, delete-on-hover), and the inline "add channel" form (name + a type dropdown).

Needs: list-row states, the popover, the inline create form.

Channel-type iconsgap

Eight channel types (text, voice/video, whiteboard, tasks, notes, reminders, docs, cubicle) currently use mismatched emoji / unicode — # 🔊 ▧ ☑ ≡ ⏰ ▤ ◻.

Needs: one coherent icon set for these 8 + the rail glyphs (desk, DM, settings).

D The channels — main-area views
Text chatused a lot

Header (channel name + "N here" presence), message list (avatar, name, time, text/image, "(edited)" tag, hover edit/delete on your own), "load older", a typing indicator, and the composer with image attach.

States: empty · someone typing · image preview before send. Needs: message row, composer, typing indicator.

Task boardthe USP · most polish

Three columns — To do / In progress / Done — each with a colored dot + count. Cards show title, assignee avatar (or a "claim" chip if unassigned), a due label, and move / delete controls. Top bar has the add-task row (title + assignee + date + Add).

States: loading · empty column · done (struck through) · overdue. This is the screen that sells the product — make it feel great. Consider how drag-and-drop could look, even though today it's arrow buttons.

Notes

A shared note that saves as you type, live for everyone. Mostly a big text surface.

Needs: the editor surface + a subtle "saved" indicator.

Reminders

A running checklist with due times.

Needs: the checklist item — checkbox, text, due time, done state.

Whiteboard

A live shared canvas; you see other people's cursors move.

Needs: the canvas frame, a small tool strip, remote-cursor labels.

Voice / video

One-click peer-to-peer call in the browser, no separate app.

Needs: pre-join state, in-call tiles (video / avatar), mute / camera / leave controls.

Embedded docs

Paste a Google Doc / Sheet / Slides link and it embeds in the channel.

Needs: the "paste a link" empty state + the embed frame / header.

Private cubicle

A space only you can see, for personal notes and drafts. Reuses the surfaces above.

Needs: a clear "this is private to you" signal.

E People & settings
DM panel

Slides out beside the rail (300px): conversation list, a "new message" flow that finds people in your teams or DMs anyone by exact @username (with a "no one with that handle" error).

Needs: the panel, conversation rows, the people-picker, the @username lookup + error.

DM chat

Same as text chat but 1:1 — no channel #.

Members

A team's member list — avatar, name, @username, role.

Needs: the member row + role tag.

Profile settings

Edit display name, @username, avatar upload.

Needs: the settings form layout + avatar uploader.

F Global pieces — design once, used everywhere
Buttons & inputs

Buttons: primary / secondary / ghost / danger, plus disabled and busy ("…"). Inputs: text, textarea, select, date, read-only — with focus + error states.

Modals & toastsreal gap

The app still uses the browser's native alert() / confirm() / prompt() for naming teams, deleting channels, and errors — ugly and off-brand.

Needs: an input dialog, a confirm / destructive dialog, and a toast (errors + "copied!").

Badges, avatars & empty states

Unread count pill, "N open" counts, the overdue treatment. Avatars: image + initials fallback (18 / 32 / 34 / 40px). Empty states: a consistent, friendly pattern across desk, channels, DMs, task columns.

04

What to hand back

The app is token-driven under the hood, so the cleaner the system you give, the faster it ships. Figma is ideal — I can pull tokens straight out. A styled mockup or annotated screenshot works too.

Surface ramp

~4 steps: app background → raised panel → card → border. Today it's a sprawl of eight near-blacks — collapse it to a clean ramp.

Text steps

High / mid / faint, plus a disabled grey. Three legible weights against every surface.

Accent not indigo

Base + hover + pressed + a soft accent-tinted background (for active rows). One accent — today there are three different blues and a stray orange.

Semantic colors

Success (done), warning (due soon), danger (overdue / delete), maybe info. Kept separate from the accent.

Type & scale

Sizes + weights for page title, section header, card title, body, small/meta, and 11px labels. Font is currently Geist — keep it or propose one.

Spacing, radii, elevation

A spacing scale (4 / 8 / 12 / 16…), corner radii (buttons vs cards vs pills vs avatars), and any shadows.

Component specs

The globals from group F — buttons, inputs, cards, badges, avatars, dialogs, toasts.

Icon set + light mode?

The 8 channel-type icons + rail glyphs, and a call on whether there's a light mode.

  3. App shell + buttons / inputs / dialogs. Rail, channel column, the global controls.
  4. Chat + the Desk. The two most-used surfaces.
  5. Everything else. Other channel types, settings, empty states.
